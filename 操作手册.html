一、系统简介

系统用途

本系统用于科技成果的评审表决，覆盖从成果录入到最终结果发布的完整流程。

核心功能

模块功能使用者
成果管理录入/编辑/删除成果、上传附件、批量导入Excel管理员
评审表决开始投票、结束投票、重置投票管理员
投票页面委员投票（同意/不同意/弃权 + 推荐等级）评委委员
结果展示查看每轮得票统计、一键发布、导出Excel管理员

系统架构

后端：Spring Boot 2.7 + H2 内嵌数据库 → 运行在 Windows 电脑上（端口 7003）

前端：Vue 2 + Element UI → 通过 Node.js 静态服务器托管（端口 3000）

平板端：安卓设备通过浏览器访问 Windows 电脑的 IP 地址，无需安装 App

关键概念：Windows 电脑作为"服务器"，安卓平板作为"客户端"。平板连接同一 WiFi 后，在浏览器输入电脑 IP 即可使用。

二、Windows 端 — 环境准备

2.1 设备要求

操作系统Windows 10 / 11（64 位）
内存≥ 4 GB（推荐 8 GB）
硬盘≥ 1 GB 可用空间
网络需连接局域网（与平板同一 WiFi）

2.2 必需文件清单

文件/文件夹说明
vote-system-windows.jar后端 JAR 包
全部前端文件index.html、server.js、options.js、*.js、*.css 等
vote-backend-windows/后端源码（仅开发需要）

2.3 需要安装的软件

1
Java 17

下载地址（任选其一）：

  • Microsoft OpenJDK 17：https://learn.microsoft.com/zh-cn/java/openjdk/download
  • Oracle JDK 17：https://www.oracle.com/java/technologies/downloads/#java17
  • Eclipse Temurin：https://adoptium.net/download/

推荐下载 MSI 安装包，双击安装即可。安装完成后打开"命令提示符"输入 java -version，看到版本号（如 17.0.x）即表示成功。

2
Node.js

下载地址：https://nodejs.org/，下载 LTS 版本安装。

安装完成后打开"命令提示符"输入 node -v，看到版本号即表示成功。

2.4 设备设置调整

  • 关闭"按电源按钮时睡眠"：控制面板 → 电源选项 → 选择"从不"
  • 关闭自动息屏：设置 → 系统 → 电源和电池 → 屏幕超时设为"从不"
  • 连接局域网：确保电脑和平板连接同一 WiFi
⚠ 重要：电脑不能在使用期间自动休眠或断网，否则平板端将无法访问系统。

三、Windows 端 — 启动系统

推荐方式：使用一键启动脚本 vote-backend-windows\启动服务.bat，双击即可自动完成：环境检测 → 启动后端 → 启动前端 → 配置防火墙 → 打开网页。

3.1 方式一：一键启动（推荐）

进入 vote-backend-windows 文件夹，双击 启动服务.bat，等待 10-15 秒即可。

脚本会自动：

  1. 检测 Java 17 和 Node.js 环境
  2. 清理旧服务进程
  3. 先后启动后端（7003）和前端（3000）
  4. 配置 Windows 防火墙放行端口
  5. 获取本机 IP 地址
  6. 自动打开浏览器访问 http://localhost:3000

启动成功后窗口显示本机和局域网平板访问地址。

3.2 方式二：手动启动命令行

在项目根目录打开 命令提示符（在文件夹空白处 Shift + 右键 → "在此处打开 PowerShell 窗口"）：

启动后端：

java -jar vote-system-windows.jar --server.port=7003

看到 Started VoteApplication 即表示成功。

启动前端（另开窗口）：

node server.js

看到 Frontend: http://localhost:3000 即表示成功。

⚠ 不要关闭弹出的命令行窗口！关闭窗口服务即停止。

3.3 停止系统

双击 vote-backend-windows\停止服务.bat，自动停止后端和前端进程。

3.4 重新构建后端

修改 Java 源码后，双击 vote-backend-windows\重新构建.bat 自动完成：停止旧服务 → 清理 → Maven 构建 → 复制 JAR 到根目录。

3.5 验证系统运行

浏览器打开 http://localhost:3000，能看到：

  • 顶部深蓝色标题栏：国网四川省电力公司科技奖励评审系统
  • 工具栏一行：搜索框 + 成果新增 + 开始投票 + 导入数据
  • 成果数据表格（含勾选框、序号、类别、名称、推荐单位、专家评审推荐、操作）

3.6 获取电脑 IP 地址

一键启动脚本会自动获取。手动方式：命令提示符输入 ipconfig，找"IPv4 地址"。

3.7 配置 options.js（平板访问必须）

编辑 options.js，将 _$base_url 改为电脑 IP：

window._$base_url = "http://192.168.1.100:7003";  // 改为实际电脑IP

四、Windows 端 — 配置防火墙

如果平板无法访问电脑，通常是 Windows 防火墙拦截了。需放行端口 3000 和 7003。

方法一：命令行快速放行（推荐）

netsh advfirewall firewall add rule name="投票系统3000" dir=in action=allow protocol=tcp localport=3000
netsh advfirewall firewall add rule name="投票系统7003" dir=in action=allow protocol=tcp localport=7003

方法二：图形界面配置

  1. 打开"控制面板" → "Windows Defender 防火墙" → "高级设置"
  2. 点击"入站规则" → "新建规则"
  3. 选择"端口" → "TCP" → "特定本地端口"填入 3000,7003
  4. "允许连接" → 全部勾选（域/专用/公用）→ 名称填写"投票系统"
如果仍无法访问，可临时关闭 Windows 防火墙测试：设置 → 隐私和安全性 → Windows 安全中心 → 防火墙和网络保护 → 关闭防火墙（测试后记得重新开启）。

五、安卓平板端 — 环境与配置

5.1 设备要求

操作系统Android 8.0 及以上
屏幕尺寸≥ 8 英寸（推荐 10 英寸及以上平板，横屏使用）
浏览器Chrome / Edge / 系统自带浏览器
网络需与 Windows 电脑连接同一 WiFi

5.2 软件准备

  1. 推荐安装 Chrome 浏览器：应用商店搜索"Google Chrome"安装
  2. PDF 阅读器：用于查看成果附件（可选，推荐 WPS Office）

  • "设置" → "WLAN"，连接与 Windows 电脑同一 WiFi
  • 关闭蓝牙和移动数据以省电
  • 建议开启"保持 WLAN 在睡眠状态下开启"

5.4 访问系统

打开 Chrome 浏览器，输入：

http://192.168.1.100:3000/tablet-vote.html

（将 IP 替换为实际电脑 IP）

六、鸿蒙平板端 — 环境与配置

本系统为鸿蒙平板提供了专用投票页面 tablet-vote.html，无需安装 App，通过浏览器即可使用。支持 PWA 安装到桌面，获得类原生应用体验。

6.1 设备要求

操作系统HarmonyOS 3.0 及以上
屏幕尺寸≥ 10 英寸平板（横屏使用，推荐 MatePad 系列）
浏览器系统自带浏览器 / 华为浏览器 / Chrome
网络需与 Windows 电脑连接同一 WiFi

6.2 软件准备

  1. 无需额外安装软件：使用平板自带的"浏览器"或"华为浏览器"即可
  2. PDF 阅读：HarmonyOS 系统内置 PDF 能力，点击附件链接自动打开

6.3 访问投票页面

打开浏览器，在地址栏输入投票专用地址：

http://192.168.1.100:3000/tablet-vote.html

将 192.168.1.100 替换为实际电脑 IP 地址。

6.4 安装到桌面（PWA）

投票页面支持 PWA 渐进式安装，添加到桌面后可以像原生 App 一样使用（全屏、无浏览器边框）。

1

在华为浏览器中打开 http://[IP]:3000/tablet-vote.html

2

点击浏览器底部工具栏的"⋮"或"更多"按钮

3

4

桌面出现图标后，点击图标即可进入全屏投票模式

6.5 投票操作说明

鸿蒙平板投票页面专为大屏触控优化：

操作方式

多台鸿蒙平板同时使用：与安卓平板相同，多台设备可同时访问投票页面，互不影响。
适配说明：投票页面采用深色科技风设计，适配 HarmonyOS 的沉浸式状态栏和横屏布局，主卡片区域支持触控滚动。

七、系统操作指南

7.1 管理员 — 录入成果

1

单个新增：点击"成果新增"按钮，弹出表单填写成果类别、成果名称、推荐单位、专家评审推荐等信息后提交。

2

批量导入：点击"导入数据"按钮，进入导入页面：

  1. 点击"下载模板"获取 Excel 模板
  2. 按模板格式填写成果信息
  3. 点击上传区域选择文件或拖拽 Excel
  4. 点击"开始导入"
系统会根据成果名称自动去重，重复的数据不会重复导入。
3

上传附件：在每条成果的操作列中，点击红色"上传附件"按钮，选择 PDF 文件上传。上传成功后按钮变为蓝色"查看附件"，点击可预览 PDF。

7.2 管理员 — 管理成果

操作说明
☑ 勾选框勾选一条或多条成果，顶部出现红色批量删除条
编辑修改成果信息
删除删除单条成果
批量删除勾选多条后点击红色"批量删除"按钮
上传附件上传 PDF 附件（红色→蓝色）
查看附件新窗口打开已上传的 PDF

7.3 管理员 — 开始投票

1

点击"开始投票"按钮，弹出确认对话框。选择是否为第一轮投票。

2

开始后，所有状态为"已提交"的成果进入投票池。

3

将投票页面地址（http:// [电脑IP]:3000/#/vote）发送给各位评委。

7.4 评委 — 投票

1

在平板浏览器打开投票地址

2

填写投票人姓名（或系统自动获取 IP）

3

逐条成果选择投票选项：同意 / 不同意 / 弃权

4

选择推荐等级：一等奖 / 二等奖 / 三等奖 / 不推荐

5

全部完成后点击提交。每人每轮只能投票一次（同一成果不可重复投票）。

7.5 管理员 — 结束投票 & 查看结果

1

所有评委投票完成后，管理员点击"结束投票"。

2

点击导航栏的"最终结果展示"，查看每项成果的投票统计：同意票数、不同意票数、弃权票数、同意占比。

3

确认无误后，点击"一键发布"将结果公开发布。

4

可点击"导出统计"下载 Excel 报表。

7.6 重置投票

如果需要重新开始投票，点击"重置"按钮。该操作会删除当前轮次的所有投票记录和结果。已发布的结果需要先撤回发布才能重置。

⚠ 警告：重置操作不可逆，请确认委员会一致同意后再执行。

八、常见问题

Q1：平板打不开系统页面（显示"无法连接"）

可能原因 & 解决：

  • 电脑和平板不在同一 WiFi → 检查两台设备的 WiFi 连接
  • 电脑防火墙拦截 → 参考第四章配置防火墙
  • 电脑 IP 地址变了 → 重新运行 ipconfig 获取新 IP
  • 后端或前端没有启动 → 检查两个黑窗口是否都在运行

Q2：点击"导入数据"没有跳转

尝试直接在浏览器输入 http:// [电脑IP]:3000/import.html 访问导入页面。

Q3：批量导入时提示"成果名称已存在"

这是正常的去重提示。同名的成果会自动跳过，不会重复导入，不计入错误。

Q4：表格表头和数据没有对齐

刷新页面即可。系统每 2 秒会自动重新对齐，如果仍然错位，按 Ctrl+Shift+R 强制刷新。

Q5：后端 JAR 启动失败

  • 检查 Java 是否已安装：java -version
  • 检查端口是否被占用：netstat -ano | findstr 7003，如有占用可换端口 --server.port=7004
  • 重置数据库：删除 %USERPROFILE%\vote_db.mv.db 文件后重启

Q6：PDF 附件上传失败

  • 确保文件格式为 .pdf
  • 文件大小不宜过大（建议 ≤ 10MB）
  • 检查后端是否正常运行

Q7：点击上传附件提示"先切换到成果管理页面"

刷新页面等待 3-5 秒让数据加载完毕后再点击。如果仍提示，强制刷新 Ctrl+Shift+R。

Q8：鸿蒙平板 PWA 安装后打开是白屏

PWA 图标可能缓存了旧地址。长按图标删除，重新在浏览器中打开 http://[IP]:3000/tablet-vote.html 后再次"添加到桌面"。

Q9：平板端投票页面提示"投票未开始"

请管理员先在 Windows 管理端点击"开始投票"。只有投票状态为"进行中"时平板端才能投票。
