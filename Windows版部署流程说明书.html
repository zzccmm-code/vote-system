📖 第0章 · 系统架构说明（先读我）

理解原理，部署不慌

这个系统是干什么的？

这是一套 2024 年度科学技术奖评审表决系统，用于评审委员在线投票表决科技奖项。替代纸质投票，实现电子化、全流程管理。

系统有三大功能模块：成果管理（录入成果信息、上传附件）、评审表决（多轮投票、实时统计）、结果展示（得票统计、最终结果发布、数据导出）。

Windows版部署是什么意思？

与"平板版"不同，Windows 版把后端程序和数据库放在 Windows 笔记本电脑上运行，前端网页仍部署在安卓平板上。电脑比平板性能更强、更稳定，适合较大规模的评审场景。

☕ 第2章 · 安装 JDK 17 运行环境

后端是 Java 程序，需要先装 Java 运行环境

2.1 检查是否已安装

在电脑上按 Win+R，输入 cmd 打开命令提示符，输入：

java -version

如果看到类似以下输出，说明已安装，可跳到第3章：

C:\Users\Administrator> java -version openjdk version "17.0.11" 2024-04-16 LTS OpenJDK Runtime Environment Microsoft (build 17.0.11+9-LTS) OpenJDK 64-Bit Server VM (build 17.0.11+9-LTS, mixed mode)

如果提示 'java' 不是内部或外部命令，说明未安装，继续下面的步骤。

2.2 下载 JDK 17

方式一：下载安装包（推荐新手，最简单）

  1. 打开浏览器，访问：
    https://learn.microsoft.com/zh-cn/java/openjdk/download
  2. 找到 "Windows x64" → "msi" 安装包，点击下载
  3. 文件名类似：microsoft-jdk-17.0.x-windows-x64.msi（约 170MB）

方式二：下载免安装版（适合不想装注册表的用户）

  1. 访问：https://aka.ms/download-jdk/microsoft-jdk-17.0.11-windows-x64.zip
  2. 下载后解压到 C:\jdk-17
  3. 手动配置环境变量（见 2.4 节）

2.3 安装 JDK（方式一用户）

  1. 双击下载的 .msi 文件
  2. 点击"下一步" → 勾选"我接受许可协议" → "下一步"
  3. 重要：在功能选择页面，确保 "Set JAVA_HOME variable" 和 "Add to PATH" 都设为"整个功能安装到本地硬盘"
  4. 点击"安装" → 等待安装完成 → "完成"
⚠️ 注意：安装过程中如果弹出"是否允许应用更改设备"，请点击"是"。

2.4 配置环境变量（仅免安装版用户需要）

如果用的是免安装版 ZIP，需要手动配置：

  1. 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
  2. 在"系统变量"下点击"新建"：
    • 变量名：JAVA_HOME
    • 变量值：C:\jdk-17（你解压的路径）
  3. 找到"系统变量"中的 Path，点击"编辑" → "新建" → 输入 %JAVA_HOME%\bin
  4. 点击"确定"保存所有窗口

2.5 验证安装

关闭之前的命令提示符窗口（环境变量修改后需要重新打开），重新打开一个新的命令提示符：

java -version

看到版本号输出即安装成功。

✅ 到这里 JDK 就装好了！这是整个部署过程中唯一需要安装的软件。

🚀 第3章 · 启动后端服务（双击即用）

JAR 包已预编译好，双击就能运行

3.1 复制项目到电脑

  1. 将 vote-backend-windows 文件夹整体复制到电脑上任意位置（推荐桌面或 D 盘根目录）
  2. 例如复制到 D:\vote-backend-windows 或 C:\Users\你的用户名\Desktop\vote-backend-windows

3.2 查看 JAR 包是否存在

打开文件夹，确认存在以下文件：

文件名说明大小
vote-system-windows.jar预编译的后端程序约 45MB
启动服务.bat启动脚本—
停止服务.bat停止脚本—
重新构建.bat修改源码后重新编译（可选）—
⚠️ 重要：如果文件夹中没有 vote-system-windows.jar 文件，需要先执行"重新构建.bat"编译（需要联网下载依赖，约10分钟）。正常情况下 JAR 包已预编译好，无需此操作。

3.3 双击启动服务

  1. 双击 启动服务.bat
  2. 会弹出一个黑色命令行窗口，显示启动日志
  3. 等待 5-10 秒，看到以下关键信息表示启动成功：
============================================ 科技奖评审表决系统 - Windows后端服务 ============================================ 正在启动后端服务... 服务地址: http://localhost:7003 H2数据库控制台: http://localhost:7003/h2-console . ____ _ __ _ _ /\\ / ___'_ __ _ _(_)_ __ __ _ \ \ \ \ ( ( )\___ | '_ | '_| | '_ \/ _` | \ \ \ \ \\/ ___)| |_)| | | | | || (_| | ) ) ) ) ' |____| .__|_| |_|_| |_\__, | / / / / =========|_|==============|___/=/_/_/_/ :: Spring Boot :: (v2.7.18) Tomcat started on port(s): 7003 (http) Started VoteApplication in 3.214 seconds
✅ 看到 "Started VoteApplication" 就说明后端启动成功了！
首次启动时会自动创建数据库和表，所以比之后启动稍慢几秒。

3.4 验证后端是否正常

在电脑浏览器中访问以下地址：

http://localhost:7003/voteRound/current

如果浏览器显示类似这样的 JSON 数据，说明后端完全正常：

{"code":200,"msg":"success","data":{"id":1,"roundNum":1,...}}

3.5 不要关闭黑窗口！

⛔ 重要：那个黑色的命令行窗口就是后端服务的运行窗口。不能关闭！关闭就等于停止了后端服务。
如果需要做其他事情，把窗口最小化即可。

3.6 如何停止后端

  • 方法一：双击 停止服务.bat
  • 方法二：在运行后端的黑窗口中按 Ctrl + C
  • 方法三：直接关闭黑窗口（不推荐，可能数据未完全保存）

3.7 查看电脑 IP 地址

后端启动后，需要查看电脑的 IP 地址，后面配置平板时要用：

  1. 按 Win+R，输入 cmd，回车
  2. 在命令提示符中输入 ipconfig，回车
  3. 找到 "无线局域网适配器 WLAN" 下的 "IPv4 地址"
C:\Users\Administrator> ipconfig 无线局域网适配器 WLAN: 连接特定的 DNS 后缀 . . . . . . . : localdomain IPv4 地址 . . . . . . . . . . . . : 192.168.1.100 子网掩码 . . . . . . . . . . . . : 255.255.255.0 默认网关. . . . . . . . . . . . . : 192.168.1.1

记下这个 IP 地址（例如 192.168.1.100），后面要用。

⚠️ 注意：如果你的电脑是通过网线连接的，找"以太网适配器"下的 IPv4 地址。确保电脑和平板在同一个 WiFi/局域网下。

🛡️ 第4章 · 配置 Windows 防火墙

让平板能够访问电脑上的后端服务

⚠️ 这一步很重要！Windows 默认会阻止其他设备访问本机的端口。如果不配置防火墙，平板将无法连接到电脑后端。

4.1 方法一：通过命令行快速放行（推荐）

  1. 按 Win+R，输入 cmd
  2. 右键点击"命令提示符"→ "以管理员身份运行"
  3. 输入以下命令（每行一个，按回车）：
# 放行 7003 端口（后端服务） netsh advfirewall firewall add rule name="VoteBackend-7003" dir=in action=allow protocol=TCP localport=7003 # 确认规则已添加 netsh advfirewall firewall show rule name="VoteBackend-7003"

看到"确定。"即表示成功。

4.2 方法二：通过图形界面配置

  1. 按 Win 键，搜索"Windows Defender 防火墙"→ 打开
  2. 点击左侧 "高级设置"
  3. 左侧点击 "入站规则" → 右侧点击 "新建规则"
  4. 选择 "端口" → "下一步"
  5. 选择 "TCP"，特定本地端口输入 7003 → "下一步"
  6. 选择 "允许连接" → "下一步"
  7. 三个网络类型（域/专用/公用）全部勾选 → "下一步"
  8. 名称输入 VoteBackend-7003 → "完成"

4.3 验证防火墙配置

在平板浏览器中输入（替换为你的电脑 IP）：

http://192.168.1.100:7003/voteRound/current

如果返回 JSON 数据，说明防火墙配置成功，平板可以访问电脑后端了。

⛔ 如果仍然打不开：
  1. 确认电脑和平板在同一个 WiFi 网络
  2. 确认电脑 IP 地址正确（ipconfig 查看）
  3. 确认后端服务正在运行（黑窗口没关）
  4. 尝试临时关闭防火墙测试：控制面板 → Windows Defender 防火墙 → "启用或关闭防火墙" → 全部关闭

📱 第5章 · 安卓平板部署前端页面

在平板上部署 Nginx 提供前端网页服务

5.1 安装 Termux

  1. 从 F-Droid（https://f-droid.org/packages/com.termux/）下载 Termux APK 并安装
  2. 打开 Termux，等待初始化完成
⚠️ 注意：不要从 Google Play 下载 Termux（版本过旧）。从 F-Droid 或 GitHub 下载。

5.2 安装 Nginx 并准备前端文件

# 更新软件包 pkg update -y # 安装 Nginx pkg install nginx -y # 创建前端目录 mkdir -p ~/vote-app/frontend

5.3 复制前端文件到平板

将项目根目录下的 5 个前端文件复制到平板的 Download 目录，然后移入 Termux：

# 复制前端文件 cp /sdcard/Download/app.*.css ~/vote-app/frontend/ cp /sdcard/Download/app.*.js ~/vote-app/frontend/ cp /sdcard/Download/chunk-vendors.*.css ~/vote-app/frontend/ cp /sdcard/Download/chunk-vendors.*.js ~/vote-app/frontend/ cp /sdcard/Download/options.js ~/vote-app/frontend/ # 查看文件 ls ~/vote-app/frontend/

5.4 创建 index.html

cat > ~/vote-app/frontend/index.html << 'HTMLEOF' 科技奖评审表决系统
HTMLEOF
⚠️ 重要：上面文件名中的哈希值必须和实际文件名完全一致。请根据你实际的文件名修改。

5.5 配置 Nginx

nano $PREFIX/etc/nginx/nginx.conf

找到 server 部分，修改为：

server { listen 8080; server_name localhost; location / { root /data/data/com.termux/files/home/vote-app/frontend; index index.html; try_files $uri $uri/ /index.html; } # 后端API反向代理到Windows电脑 location /achievement/ { proxy_pass http://192.168.1.100:7003; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } location /voteRound/ { proxy_pass http://192.168.1.100:7003; proxy_set_header Host $host; } location /voteResult/ { proxy_pass http://192.168.1.100:7003; proxy_set_header Host $host; } location /api/ { proxy_pass http://192.168.1.100:7003; proxy_set_header Host $host; } }
⛔ 关键修改：把上面所有的 192.168.1.100 替换为你的Windows 电脑实际 IP 地址！这是连接平板前端和电脑后端的关键桥梁。

5.6 启动 Nginx

# 测试配置 nginx -t # 启动 nginx # 验证 curl -I http://localhost:8080/ # 应看到: HTTP/1.1 200 OK

5.7 查看平板 IP 地址

ifconfig wlan0 | grep "inet "

记下平板 IP（例如 192.168.1.108），这是委员们要访问的地址。

5.8 设置 Nginx 开机自启（推荐）

# 需先从 F-Droid 安装 Termux:Boot mkdir -p ~/.termux/boot cat > ~/.termux/boot/start-nginx << 'EOF' #!/data/data/com.termux/files/usr/bin/sh termux-wake-lock sleep 5 nginx EOF chmod +x ~/.termux/boot/start-nginx

🌐 第6章 · 配置 options.js 连接地址

让前端知道后端在哪里

6.1 修改 options.js

nano ~/vote-app/frontend/options.js

找到文件末尾被注释掉的两行，取消注释并填入电脑的 IP 地址：

# 修改前（被注释）： // window._$base_url = "http://192.168.2.188:7003"; // window._$base_url = "/"; # 修改后（取消注释，填入你的电脑IP）： window._$base_url = "http://192.168.1.100:7003";
⛔ 最关键的一步：
  • 把 192.168.1.100 换成你的 Windows 电脑 IP（不是平板 IP！）
  • 端口 7003 不要改
  • 确保前面的 // 注释符号已经删掉

6.2 重启 Nginx 使配置生效

nginx -s reload
💡 配置总结：Windows 版中有两个 IP 需要配置：
  • Nginx 配置中的 proxy_pass → 指向电脑 IP（后端地址）
  • options.js 中的 _$base_url → 指向电脑 IP（后端地址）
  • 委员们访问的是平板 IP:8080（前端地址）

❓ 第8章 · 常见问题排查

遇到问题别慌，这里能找到答案

Q1：双击"启动服务.bat"闪退或报错？

原因：JDK 未安装或未配置环境变量。

解决：

  1. 按 Win+R → 输入 cmd → 输入 java -version
  2. 如果提示"不是内部命令"，说明 JDK 未安装，请按第2章重新安装
  3. 如果已安装但仍然报错，尝试重启电脑后再试
Q2：提示"端口 7003 已被占用"？

原因：后端服务已经在运行了，或者 7003 端口被其他程序占用。

解决：

  1. 双击 停止服务.bat 关闭旧进程
  2. 等待 2 秒后再双击 启动服务.bat
  3. 如果还不行，打开任务管理器（Ctrl+Shift+Esc），结束所有 java.exe 进程
Q3：平板打不开电脑后端（连接超时）？

排查步骤：

  1. 确认电脑和平板在同一个 WiFi 网络
  2. 确认电脑 IP 正确：ipconfig 查看
  3. 确认防火墙已放行 7003 端口（第4章）
  4. 在平板浏览器试试 http://电脑IP:7003/voteRound/current
  5. 如果还不行，临时关闭 Windows 防火墙测试
Q4：页面打开了但是数据为空或操作没反应？

原因：options.js 中的 IP 地址配置不对，或 Nginx 反向代理配置不对。

解决：

  1. 检查 options.js：window._$base_url = "http://电脑IP:7003" 确保取消注释
  2. 检查 Nginx 配置：proxy_pass http://电脑IP:7003 确保 IP 正确
  3. 执行 nginx -s reload 重新加载
  4. 在平板浏览器按 F12 → Console 查看报错信息
Q5：页面是白屏（空白页）？

原因：index.html 中的文件名和实际文件名不匹配。

解决：

  1. 在 Termux 中查看实际文件名：ls ~/vote-app/frontend/
  2. 打开 index.html，确保文件名完全一致（包括哈希值）
  3. 执行 nginx -s reload
Q6：电脑或平板重启后 IP 变了怎么办？

原因：WiFi 路由器重新分配了 IP。

解决：

  1. 查看电脑新 IP：ipconfig
  2. 修改平板 options.js 中的 IP
  3. 修改平板 Nginx 配置中的 proxy_pass IP
  4. 执行 nginx -s reload
  5. 查看平板新 IP：ifconfig wlan0
  6. 告诉委员们新的访问地址

长期方案：在路由器后台给电脑和平板绑定固定 IP（DHCP 静态分配）。

Q7：数据丢失了怎么办？

原因：H2 数据库文件在 data/ 文件夹中，删除该文件夹会丢失所有数据。

解决：定期备份 data/ 文件夹（见附录）。如果已删除，数据无法恢复，需要重新录入。

Q8：修改了源代码怎么生效？

步骤：

  1. 双击 停止服务.bat 关闭后端
  2. 双击 重新构建.bat（需要联网下载 Maven 依赖，首次约 10 分钟）
  3. 构建成功后，双击 启动服务.bat 即可

如果构建报错，检查 JDK 是否正确安装，以及网络是否通畅。

Q9：如何查看 H2 数据库里的数据？

后端启动后，在电脑浏览器访问：

http://localhost:7003/h2-console

填写连接信息：

  • JDBC URL: jdbc:h2:file:./data/vote_db;MODE=MySQL
  • User: sa
  • Password: 留空

点击"Connect"即可查看和编辑所有数据表。

Q10：能否把前端也放到电脑上运行？

可以。在电脑上安装 Nginx for Windows，把前端文件放进去，配置 8080 端口和反向代理指向 localhost:7003。这样就不需要平板了，委员们直接访问 http://电脑IP:8080。

但需要额外安装和配置 Nginx for Windows，适合有一定基础的用户。

Windows 版的所有数据都存在 vote-backend-windows/data/ 文件夹中。备份方法：

  1. 停止后端服务
  2. 将备份的 data/ 文件夹复制回来覆盖
  3. 重新启动服务
