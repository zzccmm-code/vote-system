部署前必读：整体架构说明

搞清楚"什么东西装在哪里"，后面操作不会晕

本系统分为两部分：后端服务（处理数据、保存投票）和前端页面（委员看到的网页）。两者都需要运行在安卓平板上，其他人用手机 / 电脑 / 平板的浏览器打开网址即可投票。

安卓平板
运行后端 + 前端服务
⟶
局域网 Wi-Fi
所有设备同一网络
⟶
评委手机 / 电脑
浏览器打开网址
ℹ 核心概念：安卓平板相当于一台"小服务器"。后端服务监听 7003 端口（处理数据），前端静态页面监听 8080 端口（展示界面）。评委只需在浏览器里输入平板的 IP 地址 + 端口号，就能访问。

前端页面浏览器里看到的界面8080静态 HTML/JS/CSS 文件
数据库持久化保存数据3306MySQL（随后端一起配置）
1

准备材料清单

在开始之前，确认以下东西都已就位

  • ✓安卓平板一台，Android 8.0 及以上，建议内存 4GB+，存储空间剩余 10GB+
  • ✓稳定的局域网环境（Wi-Fi 路由器），平板与所有评委设备连接同一个 Wi-Fi
  • ✓前端文件包：即本文件夹中的 app.*.css、app.*.js、chunk-vendors.*.css、chunk-vendors.*.js、options.js 五个文件
  • ✓后端 JAR 包：向系统开发方索取 vote-system.jar（或类似名称）及数据库初始化 SQL 文件
  • ✓U 盘或文件传输工具：用于将文件拷贝到平板（也可用 QQ/微信文件传输）
  • ✓蓝牙键鼠（推荐）：在平板上操作命令行时更方便，也可直接用屏幕键盘
⚠ 重要：评审当天请将平板设置为"不自动休眠"，否则服务会在屏幕熄灭后停止。路径：设置 → 显示 → 屏幕超时 → 设置为"永不"。
2

在平板上安装 Termux 终端

Termux 让安卓平板拥有"Linux 命令行"，是整个部署的基础

ℹ 什么是 Termux？可以把它理解成安卓上的"命令行工具箱"。通过它我们可以在平板上安装 Java、Nginx 等服务器软件。
  • 1

    下载 Termux 安装包

    在平板浏览器中打开下面的地址，下载最新版 Termux APK（注意：不要从 Google Play 安装，Play 上的版本已停更）：

    下载地址（在平板浏览器打开）
    https://github.com/termux/termux-app/releases/latest

# 如果 GitHub 访问慢，也可以用这个镜像地址：
https://f-droid.org/packages/com.termux/

    找到文件名类似 termux-app_v0.119.0+apt-android-7-github-debug_arm64-v8a.apk 的文件，点击下载。

  • 2

    允许安装来自未知来源的应用

    下载完成后点击安装，平板会提示"未知来源"。按照提示进入设置，打开"允许安装未知应用"，然后返回点击安装。安装完成后可以关闭该权限。

  • 3

    打开 Termux，更新软件源

    安装好后打开 Termux，会看到黑色命令行界面。依次输入以下命令（每行输完按回车）：

    Termux 终端
    # 更新软件包列表
pkg update -y

# 升级已安装的包
pkg upgrade -y
    ✓ 过程中如果出现询问"是否保留当前配置文件？"（y/n），直接按回车（默认 N，即使用新版本）即可。整个过程需要 2–5 分钟。
  • 4

    安装 Java 运行环境

    后端 JAR 包需要 Java 运行，执行以下命令安装：

    Termux 终端
    pkg install openjdk-17 -y

# 安装完成后验证（能看到版本号即成功）
java -version

    成功输出示例：openjdk version "17.0.x"

  • 5

    安装 Nginx（前端静态文件服务器）

    Termux 终端
    pkg install nginx -y

# 验证安装
nginx -v
  • 6

    安装 MySQL 数据库

    Termux 终端
    pkg install mariadb -y

# 初始化数据库目录（首次必须执行）
mysql_install_db

# 启动数据库服务
mysqld_safe &
    ✓ 最后一行命令末尾有个 &，代表"在后台运行"。输完按回车后，会看到一些提示信息，再次按回车即可回到命令行。
3

部署后端服务（Java）

配置数据库、导入数据、启动后端 JAR 包

  • 1

    将后端 JAR 包传到平板

    用 QQ/微信"文件传输助手"或 U 盘，将 vote-system.jar 和数据库初始化文件 init.sql 传输到平板，保存到"下载"文件夹。

    然后在 Termux 中，把文件复制到工作目录：

    Termux 终端
    # 在 Termux 的 home 目录创建一个专用文件夹
mkdir -p ~/vote-app/backend

# 从下载目录复制 JAR 包
cp /sdcard/Download/vote-system.jar ~/vote-app/backend/
cp /sdcard/Download/init.sql ~/vote-app/backend/
    ✓ 如果复制命令报错"Permission denied"，先在 Termux 中执行 termux-setup-storage，然后允许存储权限，再重试。
  • 2

    创建数据库并导入初始数据

    Termux 终端
    # 以 root 身份登录数据库（首次无密码，直接回车）
mysql -u root

# 以下命令在 MySQL 交互界面输入：

-- 创建数据库
CREATE DATABASE vote_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户并设置密码（把 your_password 替换成你设定的密码）
CREATE USER 'vote'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON vote_db.* TO 'vote'@'localhost';
FLUSH PRIVILEGES;

-- 退出 MySQL
EXIT;

    导入初始数据：

    Termux 终端
    cd ~/vote-app/backend
mysql -u vote -p vote_db < init.sql
# 提示输入密码时，输入刚才设置的 your_password
  • 3

    修改后端数据库连接配置

    JAR 包内通常有一个 application.yml 配置文件。如果开发方提供了外部配置文件，用以下方式编辑：

    Termux 终端
    cd ~/vote-app/backend

# 如开发方提供了 application.yml，用 nano 编辑器打开
nano application.yml

    找到数据库相关配置，改成：

    application.yml 关键配置
    # 数据库连接
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/vote_db?useUnicode=true&characterEncoding=utf8
    username: vote
    password: your_password   # 替换为你设置的密码

# 服务端口（保持 7003）
server:
  port: 7003

    编辑完成后：按 Ctrl+X → 按 Y → 按 Enter 保存退出。

  • 4

    启动后端服务

    Termux 终端
    cd ~/vote-app/backend

# 启动后端（使用外部配置文件）
java -jar vote-system.jar --spring.config.location=./application.yml > backend.log 2>&1 &

# 查看启动日志（等待约 10–20 秒）
tail -f backend.log
    ✓ 看到日志中出现 Started Application in X.XXX seconds 或 Tomcat started on port(s): 7003 字样，说明后端启动成功！按 Ctrl+C 停止日志查看（服务仍在后台运行）。
4

部署前端静态页面（Nginx）

把网页文件放到 Nginx 目录，让浏览器能打开

  • 1

    将前端文件传到平板

    把以下 5 个文件传输到平板下载文件夹（文件名中的字母数字是版本哈希，不同版本有所不同）：

    • app.*.css
    • app.*.js（去掉 .下载 后缀）
    • chunk-vendors.*.css
    • chunk-vendors.*.js（去掉 .下载 后缀）
    • options.js（去掉 .下载 后缀）
    ⚠ 文件名中的 .下载 是下载工具自动添加的临时后缀，在放入服务器前必须去掉，否则浏览器无法识别。
  • 2

    创建前端部署目录并复制文件

    Termux 终端
    # 创建前端文件目录
mkdir -p ~/vote-app/frontend/web

# 复制文件（注意：把下面的文件名替换成实际哈希值）
cp /sdcard/Download/app.5007c485.css ~/vote-app/frontend/web/
cp /sdcard/Download/app.b1193a83.js ~/vote-app/frontend/web/
cp /sdcard/Download/chunk-vendors.7eec43d6.css ~/vote-app/frontend/web/
cp /sdcard/Download/chunk-vendors.e58ab867.js ~/vote-app/frontend/web/
cp /sdcard/Download/options.js ~/vote-app/frontend/web/
  • 3

    创建 index.html 入口文件

    前端需要一个入口 HTML 文件，将所有 JS/CSS 引入进来。执行以下命令自动生成：

    Termux 终端
    cat > ~/vote-app/frontend/web/index.html <<'EOF'
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>科技奖评审表决系统</title>
  <link href="/web/chunk-vendors.7eec43d6.css" rel="stylesheet">
  <link href="/web/app.5007c485.css" rel="stylesheet">
</head>
<body>
  <div id="app"></div>
  <script src="/web/options.js"></script>
  <script src="/web/chunk-vendors.e58ab867.js"></script>
  <script src="/web/app.b1193a83.js"></script>
</body>
</html>
EOF
    ⚠ 请将上面命令中的文件名（哈希部分）替换为你实际的文件名。可以用 ls ~/vote-app/frontend/web/ 查看已复制的真实文件名。
  • 4

    配置 Nginx

    修改 Nginx 配置，将网页服务端口设为 8080，并代理后端 API：

    Termux 终端
    nano $PREFIX/etc/nginx/nginx.conf

    找到 server { ... } 代码块，替换为以下内容：

    nginx.conf 核心配置
    server {
    listen       8080;
    server_name  localhost;

    # 前端静态文件根目录
    root  /data/data/com.termux/files/home/vote-app/frontend;
    index index.html;

    # 前端路由：所有页面请求都返回 index.html
    location / {
        try_files $uri $uri/ /web/index.html;
    }

    # API 请求反向代理到后端 7003 端口
    location /api/ {
        proxy_pass http://127.0.0.1:7003/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /achievement/ {
        proxy_pass http://127.0.0.1:7003/achievement/;
        proxy_set_header Host $host;
    }

    location /voteRound/ {
        proxy_pass http://127.0.0.1:7003/voteRound/;
        proxy_set_header Host $host;
    }

    location /voteResult/ {
        proxy_pass http://127.0.0.1:7003/voteResult/;
        proxy_set_header Host $host;
    }
}

    保存：Ctrl+X → Y → Enter

  • 5

    启动 Nginx

    Termux 终端
    # 检查配置文件语法是否正确
nginx -t

# 语法正确后，启动 Nginx
nginx

# 如果已经运行，用下面命令重载配置
nginx -s reload
    ✓ nginx -t 输出 syntax is ok 和 test is successful 即代表配置正确。
5

配置 options.js 连接地址

告诉前端页面"后端服务在哪里"

ℹ 这一步是整个部署中最关键的配置。options.js 文件里的 _$base_url 变量告诉前端页面"去哪个地址找后端数据"。必须填写平板在局域网中的真实 IP 地址。
  • 1

    查询平板的局域网 IP 地址

    Termux 终端
    ifconfig
# 或者：
ip addr show

    找到 wlan0（Wi-Fi）下的 inet 后面的数字，例如 192.168.1.100，这就是平板的 IP 地址。

    ✓ 也可以在平板系统设置 → Wi-Fi → 点击已连接的网络 → 查看"IP 地址"。
  • 2

    编辑 options.js 填入 IP 地址

    Termux 终端
    nano ~/vote-app/frontend/web/options.js

    找到文件末尾这两行注释，取消注释并填入你的平板 IP：

    options.js 修改前
    // window._$base_url = "http://192.168.2.188:7003";
// window._$base_url = "/";
    options.js 修改后（将 IP 换成平板实际 IP）
    window._$base_url = "http://192.168.1.100:7003"; // ← 改成平板真实 IP
// window._$base_url = "/";  ← 保持注释

    保存：Ctrl+X → Y → Enter

  • 3

    重载 Nginx 使配置生效

    Termux 终端
    nginx -s reload

    用平板自带浏览器或同一 Wi-Fi 下的手机/电脑打开：

    浏览器地址栏输入

http://192.168.1.100:8080/web/#/achievement-upload

    能看到"成果上传"页面，说明部署成功！

    设置开机自启（可选但推荐）

    安装 Termux:Boot 插件，让数据库、后端、Nginx 在平板重启后自动运行：

    Termux 终端
    # 安装 Termux:Boot（同样从 F-Droid 下载安装）
# 安装后，创建开机启动脚本
mkdir -p ~/.termux/boot
nano ~/.termux/boot/start-vote.sh

    在文件中写入：

    start-vote.sh
    #!/data/data/com.termux/files/usr/bin/bash

# 启动数据库
mysqld_safe &
sleep 5

# 启动后端
cd ~/vote-app/backend
java -jar vote-system.jar --spring.config.location=./application.yml > backend.log 2>&1 &
sleep 10

# 启动 Nginx
nginx
    设置脚本可执行
    chmod +x ~/.termux/boot/start-vote.sh

常见问题排查

遇到问题不慌，对照下面逐一检查

Q：浏览器打开地址，显示"无法访问此网站"或"连接被拒绝"
检查以下几点：
1. 平板和访问设备是否在同一个 Wi-Fi？
2. Nginx 是否已启动？在 Termux 执行 nginx -t && nginx
3. IP 地址是否填写正确？用 ifconfig 重新确认
Q：页面能打开，但显示"加载失败"或数据为空
后端服务可能未启动或已崩溃。执行 tail -f ~/vote-app/backend/backend.log 查看日志，检查是否有报错信息。常见原因：数据库密码错误、端口被占用。
Q：文件名报错，提示找不到 JS/CSS 文件
index.html 中引用的文件名与实际文件名不一致。执行 ls ~/vote-app/frontend/web/ 查看真实文件名，然后用 nano ~/vote-app/frontend/web/index.html 修正文件名。
Q：平板重启后，服务消失了
需要重新手动启动，或按照步骤 6 中的"开机自启"设置 Termux:Boot 脚本。
Q：Termux 安装软件时提示"Storage permission denied"
在 Termux 中执行 termux-setup-storage，然后在弹出的权限请求中点击"允许"。
Q：评委反映投票页面打开后转圈圈，无法加载
通常是 options.js 中的 _$base_url 地址填写有误。检查：① IP 是否是平板当前 IP（IP 可能因重新连接 Wi-Fi 而变化）；② 端口是否为 7003；③ 地址前缀是否为 http://（不是 https）。
Q：平板 Wi-Fi IP 每次连接都会变，评委要重新记地址？
建议在路由器后台为平板设置"固定 IP/静态地址绑定"（根据 MAC 地址绑定）。具体操作参考路由器品牌的说明书，一般在"LAN 设置" → "地址保留"中配置。
Q：系统时区或时间显示不对
在 Termux 执行：export TZ=Asia/Shanghai，然后重启后端服务。
