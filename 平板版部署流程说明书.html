📖 第0章 · 系统架构说明（先读我）

理解原理，部署不慌

这个系统是干什么的？

这是一套 2024 年度科学技术奖评审表决系统，用于评审委员在线投票表决科技奖项。替代纸质投票，实现电子化、全流程管理。

系统有三大功能模块：成果管理（录入成果信息、上传附件）、评审表决（多轮投票、实时统计）、结果展示（得票统计、最终结果发布、数据导出）。

平板版部署是什么意思？

就是把 前端页面 + 后端程序 + 数据库 全部装进一台安卓平板里，平板充当一台小型"服务器"。评审委员们用自己的手机或平板通过浏览器访问这台"平板服务器"进行投票。

1.4 平板设置调整

  1. 关闭省电模式：设置 → 电池 → 关闭省电模式 → 将 Termux 加入"不受限制"应用列表

  4. 安装 Termux:Boot（可选）：从 F-Droid 安装，用于开机自动启动服务

🔧 第2章 · 安装 Termux 运行环境

在平板上创建一个"迷你Linux"环境

2.1 安装 Termux

  1. 从 F-Droid（https://f-droid.org/packages/com.termux/）下载 Termux APK
  2. 安装后打开，会看到一个黑色命令行界面
  3. 第一次打开需要等几秒钟初始化

2.2 更新软件包并安装核心工具

在 Termux 中依次输入以下命令（每行输完按回车）：

# 1. 更新软件源（如果很慢可以跳过） pkg update -y && pkg upgrade -y # 2. 安装核心工具 pkg install openssh nano wget curl unzip -y # 3. 安装 Java 17（后端运行需要） pkg install openjdk-17 -y # 4. 安装 Maven（编译后端代码需要） pkg install maven -y # 5. 安装 MariaDB（数据库，替代 MySQL） pkg install mariadb -y # 6. 安装 Nginx（提供前端网页） pkg install nginx -y

2.3 验证安装

# 检查 Java java -version # 应输出: openjdk version "17.x.x" # 检查 Maven mvn -version # 应输出: Apache Maven 3.x.x # 检查 MariaDB mysql --version # 应输出: mariadb from xx.x.x-MariaDB # 检查 Nginx nginx -v # 应输出: nginx version: nginx/x.x.x
💡 小知识：Termux 就是一个安卓上的"迷你 Linux"。你在这里输入的命令，和在电脑 Linux 终端里输入的几乎一样。

2.4 创建项目目录

# 在 Termux 主目录下创建项目文件夹 mkdir -p ~/vote-app/vote-backend mkdir -p ~/vote-app/frontend # 查看目录 ls ~/vote-app/ # 应看到: vote-backend frontend

💾 第3章 · 安装并配置 MariaDB 数据库

数据库是存投票数据的地方

3.1 启动 MariaDB

# 启动数据库服务 mysqld_safe & # 等待 3-5 秒，然后测试连接 mysql -u root
💡 说明：Termux 里的 MariaDB 不需要密码，直接用 mysql -u root 就能进去。如果看到 MariaDB [(none)]> 提示符，说明成功了。

3.2 创建数据库和用户

在 MariaDB 命令行里输入以下 SQL：

-- 创建数据库 CREATE DATABASE vote_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; -- 创建用户并授权 CREATE USER 'vote'@'localhost' IDENTIFIED BY 'vote123456'; GRANT ALL PRIVILEGES ON vote_db.* TO 'vote'@'localhost'; FLUSH PRIVILEGES; -- 退出 EXIT;

3.3 导入建表脚本

首先需要把 init.sql 文件复制到 Termux 目录。假设你已通过文件管理器将文件放在了平板的下载目录：

# 复制 init.sql 到项目目录 cp /sdcard/Download/init.sql ~/vote-app/vote-backend/ # 导入数据库 mysql -u vote -pvote123456 vote_db < ~/vote-app/vote-backend/init.sql # 验证表是否创建成功 mysql -u vote -pvote123456 vote_db -e "SHOW TABLES;"

如果看到 4 张表（achievement、vote_round、vote_result、vote_record），说明数据库准备完毕。

# 预期输出： +-------------------+ | Tables_in_vote_db | +-------------------+ | achievement | | vote_record | | vote_result | | vote_round | +-------------------+

3.4 设置数据库开机自启（推荐）

# 创建自启脚本 mkdir -p ~/.termux/boot cat > ~/.termux/boot/start-mariadb << 'EOF' #!/data/data/com.termux/files/usr/bin/sh termux-wake-lock mysqld_safe & EOF chmod +x ~/.termux/boot/start-mariadb

需要先安装 Termux:Boot 应用才能生效。

⚙️ 第4章 · 部署后端 JAR 服务

后端是系统的"大脑"，处理所有投票逻辑

4.1 复制后端项目文件

将 vote-backend 整个文件夹复制到平板，然后移入 Termux 目录：

# 假设你已将 vote-backend 文件夹复制到平板的 Download 目录 cp -r /sdcard/Download/vote-backend/* ~/vote-app/vote-backend/ # 验证文件 ls ~/vote-app/vote-backend/ # 应看到: pom.xml README.md src/ ...

4.2 编译后端 JAR 包

⚠️ 注意：编译过程需要联网下载依赖（约 200MB），首次编译可能需要 10-20 分钟，请保持网络畅通和耐心。
# 进入项目目录 cd ~/vote-app/vote-backend # 开始编译（跳过测试） mvn clean package -DskipTests

编译成功后会看到：

... [INFO] BUILD SUCCESS [INFO] Total time: xx:xx min ... # 验证 JAR 文件已生成 ls -lh target/vote-system-1.0.0.jar # 应看到约 45MB 的 jar 文件

4.3 启动后端服务

# 后台启动后端服务（端口 7003） cd ~/vote-app/vote-backend nohup java -jar target/vote-system-1.0.0.jar > ~/vote-app/backend.log 2>&1 & # 等待 8-10 秒让服务启动 sleep 10 # 检查是否启动成功 curl -X POST http://localhost:7003/voteRound/current # 应返回: {"code":200,"msg":"success","data":...}
💡 如果返回了 JSON 数据，说明后端启动成功！如果返回"Connection refused"，查看日志：cat ~/vote-app/backend.log

4.4 检查启动日志

# 查看后端日志最后 20 行 tail -20 ~/vote-app/backend.log # 看到以下内容说明启动成功： # Tomcat started on port(s): 7003 (http) # Started VoteApplication in x.xxx seconds

4.5 设置后端开机自启（推荐）

cat > ~/.termux/boot/start-backend << 'EOF' #!/data/data/com.termux/files/usr/bin/sh sleep 5 cd ~/vote-app/vote-backend nohup java -jar target/vote-system-1.0.0.jar > ~/vote-app/backend.log 2>&1 & EOF chmod +x ~/.termux/boot/start-backend

🎨 第5章 · 部署前端页面（Nginx）

前端就是委员们看到的投票网页

5.1 准备前端文件

将项目根目录下的 5 个前端文件复制到 Termux：

# 复制前端文件到 Nginx 目录 cp /sdcard/Download/app.*.css ~/vote-app/frontend/ cp /sdcard/Download/app.*.js ~/vote-app/frontend/ cp /sdcard/Download/chunk-vendors.*.css ~/vote-app/frontend/ cp /sdcard/Download/chunk-vendors.*.js ~/vote-app/frontend/ cp /sdcard/Download/options.js ~/vote-app/frontend/ # 查看文件 ls ~/vote-app/frontend/

5.2 创建 index.html 入口文件

前端需要一个 index.html 作为网页入口。在 ~/vote-app/frontend/ 目录下创建：

cat > ~/vote-app/frontend/index.html << 'HTMLEOF' 科技奖评审表决系统
HTMLEOF
⚠️ 重要：上面 <script> 和 <link> 标签里的文件名必须和实际文件名完全一致（包括那串哈希值）。请根据你实际下载的文件名修改。

5.3 配置 Nginx

# 编辑 Nginx 配置文件 nano $PREFIX/etc/nginx/nginx.conf

找到 server 部分，修改为以下内容：

server { listen 8080; server_name localhost; # 前端静态文件 location / { root /data/data/com.termux/files/home/vote-app/frontend; index index.html; try_files $uri $uri/ /index.html; } # 后端 API 反向代理 location /achievement/ { proxy_pass http://127.0.0.1:7003; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } location /voteRound/ { proxy_pass http://127.0.0.1:7003; proxy_set_header Host $host; } location /voteResult/ { proxy_pass http://127.0.0.1:7003; proxy_set_header Host $host; } location /api/ { proxy_pass http://127.0.0.1:7003; proxy_set_header Host $host; } }

保存退出（Nano 编辑器：按 Ctrl+O 保存，Enter 确认，Ctrl+X 退出）。

5.4 启动 Nginx

# 测试配置是否正确 nginx -t # 应输出: configuration file ... test is successful # 启动 Nginx nginx # 验证前端是否可访问 curl -I http://localhost:8080/ # 应看到: HTTP/1.1 200 OK

5.5 设置 Nginx 开机自启（推荐）

cat > ~/.termux/boot/start-nginx << 'EOF' #!/data/data/com.termux/files/usr/bin/sh sleep 8 nginx EOF chmod +x ~/.termux/boot/start-nginx

🌐 第6章 · 配置 options.js 连接地址

让前端知道后端在哪里

6.1 查看平板 IP 地址

# 方法一：Termux 命令查看 ifconfig wlan0 | grep "inet " # 或 ip addr show wlan0 | grep "inet " # 方法二：平板设置查看 # 设置 → WiFi → 点击已连接的WiFi → 查看 IP 地址

记下这个 IP 地址，例如：192.168.1.108

6.2 修改 options.js

# 编辑 options.js 文件 nano ~/vote-app/frontend/options.js

找到文件末尾被注释掉的那行，取消注释并填入平板 IP：

# 修改前（被注释）： // window._$base_url = "http://192.168.2.188:7003"; // window._$base_url = "/"; # 修改后（取消注释，填入你的平板IP）： window._$base_url = "http://192.168.1.108:7003";
⚠️ 关键：把 192.168.1.108 换成你实际查到的平板 IP！端口 7003 不要改。

6.3 重启 Nginx 使配置生效

# 重新加载 Nginx 配置 nginx -s reload

❓ 第8章 · 常见问题排查

遇到问题别慌，这里能找到答案

Q1：其他手机打不开平板上的系统页面？

排查步骤：

  1. 确认手机和平板连的是同一个 WiFi
  2. 确认平板 IP 地址正确（ifconfig wlan0 查看）
  3. 确认 3 个服务都在运行：ps aux | grep -E "mysqld|java|nginx"
  4. 在手机浏览器试试 http://平板IP:7003/voteRound/current，如果返回 JSON 说明后端没问题，问题在前端
  5. 如果都打不开，检查平板是否开启了防火墙（一般安卓没有）
Q2：后端启动报错 "Connection refused" 或数据库连接失败？

原因：MariaDB 数据库没有启动，或用户名密码不对。

解决：

  1. 先启动数据库：mysqld_safe &
  2. 等待 3 秒再启动后端
  3. 检查用户名密码：在 application.yml 中确认 username: vote、password: vote123456
  4. 查看后端日志：cat ~/vote-app/backend.log | tail -50
Q3：mvn 编译很慢或报错下载失败？

原因：Maven 默认从国外服务器下载依赖，网络不通或太慢。

解决：配置国内镜像源。编辑 ~/.m2/settings.xml：

<settings>
  <mirrors>
    <mirror>
      <id>aliyun</id>
      <mirrorOf>central</mirrorOf>
      <name>Aliyun Maven</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>

然后重新执行 mvn clean package -DskipTests

Q4：页面打开了但是是空白页？

原因：index.html 里的文件名和实际文件名不一致。

解决：

  1. 在 Termux 中查看实际文件名：ls ~/vote-app/frontend/
  2. 打开 index.html，把 <script> 和 <link> 标签中的文件名改为实际文件名
  3. 执行 nginx -s reload 重新加载
Q5：投票数据提交后看不到结果？

排查步骤：

  1. 确认 options.js 中的 IP 地址正确且已取消注释
  2. 在浏览器按 F12 打开开发者工具 → Console 面板，看是否有红色报错
  3. 检查 Network 面板，看 API 请求是否返回 200
  4. 如果返回 404，检查 Nginx 反向代理配置是否正确
Q6：平板重启后服务都没了？

原因：没有设置开机自启。

解决：

  1. 安装 Termux:Boot 应用（从 F-Droid 下载）
  2. 按照第3/4/5章中的自启脚本配置
  3. 重启平板验证是否自动启动

或者手动启动，参考第7章 7.3 节的完整启动命令。

Q7：平板 IP 地址变了怎么办？

原因：WiFi 路由器重新分配了 IP。

解决：

  1. 查看新 IP：ifconfig wlan0
  2. 修改 options.js 中的 IP 地址
  3. 执行 nginx -s reload
  4. 告诉委员们新的访问地址

长期方案：在路由器后台给平板绑定固定 IP（DHCP 静态分配）。

Q8：如何备份数据？

方法一：导出 SQL

mysqldump -u vote -pvote123456 vote_db > /sdcard/Download/vote_backup.sql

方法二：备份整个数据库文件

cp -r $PREFIX/var/lib/mysql/vote_db /sdcard/Download/vote_db_backup/

建议每天评审结束后备份一次！
